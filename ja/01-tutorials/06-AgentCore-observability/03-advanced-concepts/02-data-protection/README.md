# Amazon Bedrock AgentCore オブザーバビリティ: データ保護

## 概要

このチュートリアルでは、Amazon Bedrock Guardrails と Amazon CloudWatch Logs データ保護ポリシーを使用して、エージェント AI アプリケーションに包括的なデータ保護を実装する方法を学びます。このチュートリアルでは、入力処理から出力生成、ログ記録まで、エージェントのライフサイクル全体を通じて機密データを保護する方法を示します。

個人を特定できる情報（PII）、財務データ、健康記録、その他の機密情報を保護するために連携して動作する複数の保護レイヤーを組み合わせることで、AI アプリケーションを保護するための多層防御戦略の作成に焦点を当てます。

### チュートリアルアーキテクチャ

このチュートリアルでは、AgentCore ランタイムにデプロイされたエージェントに対するデータ保護メカニズムの実装方法を示します。機密情報を処理するカスタマーサポートエージェントを使用し、複数のセキュリティレイヤーを使用してこのデータを保護する方法を示します。

サンプルには以下が含まれます：
- カスタマーサポート機能を備えた Strands Agent
- コンテンツフィルタリング用の Amazon Bedrock Guardrails
- ログマスキング用の CloudWatch Logs データ保護
- 機密情報の検出と処理

### チュートリアルの主な機能

* **多層データ保護**: Bedrock Guardrails と CloudWatch Logs データ保護の実装
* **機密情報検出**: PII、財務データ、その他の機密情報の自動検出
* **エージェントセキュリティ**: エージェントインタラクションとトレースにおける機密データの保護
* **コンプライアンスサポート**: プライバシー規制（GDPR、HIPAA、CCPA）要件への対応
* **多層防御戦略**: エージェント AI アプリケーションの包括的なセキュリティの作成

## 学習内容

このハンズオンチュートリアルでは、以下を探索します：

- エージェントインタラクションと CloudWatch Logs およびトレースで機密情報を検出する方法
- Amazon Bedrock Guardrails: AI エージェントが機密コンテンツを処理または生成することを防ぐための機密情報フィルターの設定方法
- CloudWatch Logs データ保護: アプリケーションログで機密データを自動的に検出してマスクする方法
- AgentCore 統合: エージェントワークフロー内でこれらの保護対策を実装する方法

## なぜこれが重要か

適切な保護策がなければ、エージェント AI システムは以下のことが起こり得ます：

- レスポンスやログで意図せず機密顧客データを露出
- プライバシー規制に違反する情報の処理または保持
- 共有すべきでない PII を含む出力の生成
- アプリケーションインフラストラクチャにコンプライアンスとセキュリティの脆弱性を作成
